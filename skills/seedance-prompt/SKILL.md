# Seedance 女装千川提示词

## Overview

把女装商品图或简单想法转成 Seedance 图生视频提示词，并附可用于千川/抖音信息流的短视频文案建议。默认服务女装，也兼容其他服装类商品。目标是让画面像真实拍摄的人物展示：自然、有呼吸感、低 AI 感、适合投放测试。

## Workflow

1. 判断输入是图生视频还是文生视频。默认按图生视频处理；只有用户明确说文生视频时才切换。
2. 提取或补全：品类、服装卖点、穿着场景、人物动作、镜头、风格、是否需要文案、是否要多方向。
3. 缺少“生成时长”或“内容/动作”时，必须先问用户，不直接生成最终提示词。问题保持简短，例如：“要几秒？想做什么动作/内容？” 只有这两项明确后再写可复制提示词。
4. 默认输出 1 个可直接复制到 Seedance 的精修提示词。用户要求多个方向、批量测款、5个版本等时，输出多个可复制提示词；只有用户明确说“完整报告”“反推报告”“镜头任务清单”时，才输出报告结构。
5. 如果用户提供多模态素材，先为每个素材分配明确职责：人物/服装/商品、首帧、尾帧、背景、参考视频动作、参考视频镜头、参考视频节奏、参考音频/BGM/音色。不要只写“参考 @Video1”。
6. 先做“五维扫描”，只补素材没有提供的维度；素材已经锁死的人物、服装、背景和风格不要反复铺写。
7. 默认输出克制版提示词：用 1 个素材信息块、1 个素材锁定块、1 个画面执行块、1 个秒级剧本时间轴、1 个“避免项（以下全部不要生成，不是画面内容）”块。只有高风险细节才展开。
8. 可复制提示词必须给出生成时长和秒级时间轴。用户未提供时长或内容/动作时，不要自行编剧本，不要默认 6 秒；先向用户确认。参考视频复刻按原片或拆镜头时长执行。

## Default Output Contract

默认只输出可复制提示词，不输出完整报告。每条提示词必须包含：

1. `建议生成时长`
2. `人物@，服装@`
3. 人物站位/构图
4. 秒级剧本时间轴
5. 口播时间轴
6. 手机真实拍摄质感
7. 避免项（以下全部不要生成，不是画面内容）

如果用户只说“帮我写一条”“根据某款号写提示词”但没有给时长和动作/内容，停止在澄清问题，不输出提示词。必须问：“要几秒？想做什么动作/内容？” 如果用户只缺其中一项，只问缺少的那一项。

不要默认输出完整报告、背景空图任务、镜头任务清单、千川文案或普通“5条创意提示词”列表。只有用户明确要求普通创意、批量方向、千川文案或广告标题时，才使用旧的创意/文案结构。只有用户明确要求完整复刻报告时，才输出 `统一视觉设定 / 素材信息 / 背景空图生成任务 / 镜头任务清单 / 原动作版完整复制区`。

开头引用行保持极简，只写 `人物@，服装@`，不要写 `Image1`、`Image2`、`ImageN`、`ImageM`，也不要在 `人物@` 后面追加款号、人物数量、人物外观、服装信息等长字段。人物身份由 `人物@` 锁定，服装款式和细节由 `服装@` 锁定。后文只写动作、站位、构图、时间轴和稳定要求。

禁止输出只有多个短 `text` 代码块且缺少时间轴/人物信息的列表。每个版本都必须是完整可复制提示词。

面向千川/投流/广告素材时，禁止只写“走两步、转身、整理衣服、定格”这类普通展示。每条提示词必须先有投流脚本骨架：前 1-3 秒画面钩子、一个明确买家任务、一个真实场景阻力、至少两段可见卖点证明、一个结尾记忆点。动作必须服务于卖点证明，而不是为了让人物动起来。

面向千川/投流/广告素材时，必须显式选择 1 个钩子类型，并把它落成可拍画面，不要只写一句抽象文案。钩子类型写进创意定位或千川文案建议，不写进 Seedance 画面文字。

默认使用这个结构：

```text

```

普通创意输出仅在用户明确要求时使用：

```markdown

**Seedance 图生视频提示词**
**负面提示词**
**参数建议**

```

如果用户说“只要提示词”“不要文案”，省略千川文案，不改变复刻报告结构。如果用户要求多版本，每个版本都保持完整但不堆砌；多版本之间必须更换买家任务、场景阻力、画面钩子和卖点证明，不能只更换背景或动作名称。

## Prompt Rules

### 画面证据写法

Seedance 提示词必须优先写可见画面证据，少写抽象词和否定句：

- 不要用“高级感、松弛感、电影感、治愈感、氛围感、原生感、故事感”等空泛词替代画面描述。如果必须使用抽象词，只能作为辅助，前后必须有具体画面证据支撑。
- 写清具体时间、地点、人物姿态、手部动作、眼神、光线方向、服装细节、背景物体、镜头距离和人物站位。
- 把抽象表达改成可见画面：不要只写“治愈感女生故事”，要写“清晨公交站，女生站在站牌右侧，阳光从画面左上方照亮白衬衫，头发被风吹向画面右侧，远处有车流和行人虚化经过”。
- 主提示词优先使用肯定结果，少写“不要下雨、不要夸张表情、不要动漫风”，改写为“晴天后阳光照在地面，空气清透”“表情平静，嘴唇自然闭合，眼神放松”“真实摄影质感，自然皮肤纹理，真实镜头光影”。
- 少写“创业、约会、直播、庆祝、通勤、日常、精致、大片”等容易触发固定套路的模板词。必须落到具体空间和物体，例如“凌晨小房间，折叠桌上有泡面盒、笔记本电脑和手写草稿，人物坐在桌前低头修改方案”。
- 女装场景也必须具体：不要只写“通勤感”，改写为“电梯口灰色墙面前，人物左手拿帆布包，右手整理袖口，脚边有浅色地砖线”。
- 负面约束只放在末尾“避免项（以下全部不要生成，不是画面内容）”中，并用“避免生成/不要生成”开头；主提示词尽量写正向画面结果。

### 五维扫描和素材锁定

写提示词前先做五维扫描，判断素材/剧本已经锁定什么、还缺什么：

1. 绝对主体与物理动势：人物/商品、动作、姿态、站位。
2. 环境场与情绪光影：场景、光线方向、色调、空间锚点。
3. 光学与摄影机调度：景别、焦距感、推拉摇移、环绕、跟拍。
4. 时间轴与状态演变：动作如何从起始状态过渡到结果状态。
5. 美学介质与底层渲染：真实手机感、胶片感、油画感、电影光影、材质质感。

素材已经锁定的维度不要重复铺写。图片通常锁人物、服装、商品、首帧、背景和部分风格；视频通常锁动作、镜头、节奏和表情变化；音频通常锁 BGM、卡点、口播音色和情绪。信息越满，提示词越克制；信息越少，才展开补主体、动作、场景、镜头、时间轴和质感。只对空白维度精准补充，避免文字特征和素材特征抢注意力。

输出前做注意力污染检查：颜色、材质、光效、道具和风格词必须绑定到正确对象，例如“暗红色腰带”不能让红色跑到天空、背景或服装其他区域。抽象风格词必须转成光线、焦距、景别、材质、色彩、空间和动作证据。

### 提示词压缩原则

用 MyElc 式五维控制思路优化提示词：不要套万能长模板；先判断素材已经给了哪些参数，再只对空白参数下指令。

- 四视图/参考图已经锁定的人物身份、脸、眼睛、发型、妆容、身材比例、服装、logo/印花/纽扣/拉链/口袋/走线/装饰件和材质，默认合并成 1-2 句“保持一致”锁定块，不要逐项正写一遍再负写一遍。
- 主提示词不要反复写“不换脸、不漂移、不变化”。这些只放进末尾避免项，除非用户明确说某个部位经常失败。
- 优先把字数给空白维度：场景空间、人物站位、光线方向、镜头方式、动作时间轴、自然停顿和结束姿态。
- 图生视频素材信息很满时，提示词要克制；例如输入图已锁主体、服装、背景和风格，只补“镜头怎么动”和“人物做什么轻动作”。
- 文生视频或素材很少时，才展开补主体、动作、空间、光线、镜头、时间轴和美学介质。
- 用户提供臃肿提示词要求优化时，先去重：保留素材锁定、站位构图、场景光线、时间轴、手机质感、负面硬错误；删除重复的同义约束和连续否定句。
- 复杂度超过 8 秒且包含 4 个以上主要动作时，建议拆成 2 条视频；若用户坚持一条，减少动作数量或降低镜头运动。

推荐克制结构：

```text
素材锁定：保持参考图/人物四视图中的同一人物身份、脸型、五官、眼型、眼距、眼神气质、发型、妆容、肤色方向、身高比例、胖瘦比例和身材比例；保持服装颜色、版型、logo/印花/纽扣/拉链/口袋/走线/装饰件、面料材质、纹理、厚薄、垂感、光泽、褶皱位置和拼接结构一致。
画面执行：写清画幅、站位、脚底落点、身体中心线、头顶/脚底留白、背景锚点、光线方向、手机拍摄质感。
剧本时间轴：按秒拆动作，每段只写一个主要动作和一个镜头目标。
口播时间轴：无可识别口播时写“不生成口播台词”。
避免项（以下全部不要生成，不是画面内容）：避免生成硬错误。
```

### 硬优先级

生成 Seedance 提示词时，优先级固定为：

1. 人物一致性
2. 脸型和眼神一致性
3. 人物站位一致性
4. 服装 logo 和细节 1:1 一致
5. 服装质感一致性
6. 服装颜色、版型、面料、花纹、领口、袖口、腰线、下摆一致
7. 动作和镜头自然
8. 背景和手机拍摄质感稳定
9. 多模态 @ 引用职责明确
10. 时长、动作复杂度、镜头指令不冲突
11. 广告钩子和购买心理匹配

如果这些目标冲突，优先保住人物四视图/参考图中的同一人物身份、同一张脸、脸型、眼型、眼距、眼神气质、下颌线、颧骨、下巴、鼻梁、嘴型、发型、妆容、肤色方向和身材比例，其次保住服装 logo、印花、纽扣、拉链、口袋、走线、装饰件、颜色、版型、面料材质、织物纹理、厚薄、垂感、光泽、透明度、弹性、硬挺度、花纹、领口、袖口、腰线、下摆和褶皱位置。

### Seedance 2.0 多模态引用规则

使用 Seedance 2.0 的 `@Image` / `@Video` / `@Audio` 时，每个引用必须写清用途：

- `人物@`：人物身份、脸、眼神、身材、发型、妆容、肤色方向参考。
- `服装@`：服装/商品主体、颜色、版型、面料、logo、印花和细节参考。
- `视频@`：只参考动作、镜头运动、节奏、转场、特效、表情变化或场景氛围中的明确一项或几项。
- `音频@`：参考 BGM 节奏、音色、口播情绪、环境声或音效。
- 多素材时按稳定性分工：图片锁人物/服装/商品，视频锁动作/镜头/节奏，音频锁声音/卡点。
- 禁止写 `@Image1`、`@Image2`、`@Video1` 这类编号引用。需要视频或音频参考时，写成“视频@ 只参考镜头推进和动作节奏；人物身份以人物@为准，服装款式以服装@为准”。

约束检查：图片不超过 9 张，视频不超过 3 个，音频不超过 3 个，总素材不超过 12 个；参考视频/音频优先控制在 2-15 秒。若用户素材超限，先帮用户筛选最影响画面的素材。

### 剧本驱动时间轴和复杂度

用户提供时长、剧本、参考视频，或要求 8 秒以上视频时，先读剧本/参考动作，再拆动作节拍，不要机械套固定的 `0-3s / 3-6s / 6-9s`。时间轴必须跟剧情和动作逻辑走：

- 先拆出起始状态、触发动作、主要变化、结果状态；剧情短片还要拆人物意图、动作推进、情绪变化、结果。
- 再按目标总时长和动作难度分配每段时间；简单动作可短，复杂转身、变身、走位、特效生长要给更长过渡。
- 每段只写一个主要动作和一个主要镜头目标。人物大动作时镜头要稳；镜头大运动时人物动作要简单。
- 每段必须写清人物位置、手部动作、头部/眼神、身体重心、服装/商品变化、镜头运动、自然停顿、光线和背景物体、结束姿态。
- 避免同一段里同时转身、换景、爆炸、推镜头、切近景、换表情。复杂动作要拆段，不能硬塞进 4-5 秒。

常见节拍拆法：服装展示按“站定、抬手/整理、转身/走动、定格”拆；变身/特效按“触发、能量出现、结构生长、完成”拆；剧情短片按“意图、推进、情绪变化、结果”拆。女装千川默认保持单场景、单人物、单主动作，优先稳定和低 AI 感。

做参考视频反推或用户要求“剧本”时，每条可复制 Seedance 提示词内部必须包含“剧本时间轴”。时间段按参考视频实际节奏或剧本节拍拆分，不固定死。不要只写泛泛的“自然展示”或抽象情绪。

如果需要口播，每条可复制 Seedance 提示词内部必须包含“口播时间轴”。只有参考视频有可识别口播时才写具体台词；没有口播时写明不生成口播台词；听不清时只标注说话时间段，不编台词。口播可以生成声音/口播音频，也可以作为口型节奏参考；人物嘴部只做自然轻微开合，不要夸张对口型；画面内一律禁止字幕、口播文字、屏幕文字、手机界面文字、UI文字和任何可读文字。

### 镜头与声音

默认使用手机真实拍摄感：固定镜头、轻微推进、轻微横移、跟随半步、自然中近景。只有用户明确要电影感/广告片/MV 时，才使用更强镜头词，如推镜、拉镜、环绕、跟拍、俯拍、低角度、快速甩镜、一镜到底。

不要让镜头指令互相冲突：不要同时写“固定镜头”和“环绕拍摄”，不要同时写“近景锁定”和“大幅拉远全身”。如需要音频方向，可写 BGM 氛围、节奏、衣料摩擦、脚步声、室内环境声；不要让声音描述抢过画面稳定性。

### 图生视频默认写法

图生视频提示词必须锚定参考图，但默认使用压缩锁定块，不要把人物和服装细节拆成多段重复长约束：

```text
保持参考图/人物四视图中的同一人物身份、脸型、五官、眼型、眼距、眼神气质、发型、妆容、肤色方向、身高比例、胖瘦比例和身材比例。保持服装颜色、版型、logo/印花/纽扣/拉链/口袋/走线/装饰件、面料材质、纹理、厚薄、垂感、光泽、褶皱位置和拼接结构一致；只添加自然真实的轻微动作和镜头变化：...
```

重点保持服装不跑偏：颜色不变、版型不变、面料质感不变、花纹、logo、印花、纽扣、拉链、口袋、走线、装饰件、腰带、褶皱位置不乱变。只增加真实拍摄中会出现的微小变化。

### 人物一致性

每条 Seedance 提示词开头必须先写人物一致性，但默认压缩成一句，不要连续堆叠同义否定词：

- 保持参考图/人物四视图中的同一人物身份、同一张脸、同一脸型、同一下颌线、同一颧骨、同一下巴、同一鼻梁、同一嘴型、同一五官、同一发型、同一妆容、同一肤色方向、同一身高比例、同一胖瘦比例、同一身材比例。
- “不换脸、不脸型漂移、不下颌线变化”等负面内容默认放进负面提示词，不在主提示词里反复展开。
- 肤色可以写“白净自然，气色干净，保留真实皮肤纹理”，但不要过度磨皮。

### 脸型和眼神一致性

每条 Seedance 提示词必须写眼部一致性；默认合并到人物锁定块，只有眼神/眼型是高风险时才单独展开：

- 保持参考图/人物四视图中的同一眼型、同一眼距、同一双眼大小比例、同一眼皮形态、同一瞳孔比例、同一眼神气质。
- 视线方向可以跟随动作自然移动，但必须有明确落点。
- 眼神发散、视线错位、斜视、斗鸡眼、大小眼、闭眼异常、眨眼畸形、瞳孔异常、空洞眼神等放进负面提示词。

### 人物站位一致性

提示词必须写清人物站位，尤其是做参考视频复刻或有背景图时：

- 人物位于画面左/中/右哪个区域。
- 脚底落点在画面底部几分之几处。
- 身体中心线在画面横向几分之几处。
- 头顶留白、脚底留白、人物占画面高度比例。
- 人物相对门框、墙角、窗户、扶手、地砖线、地面反光、镜子边缘、沙发边缘等背景参照物的位置。
- 镜头推进、后退、下移或跟拍时，脚底落点不漂移，身体中心线不乱跳，与背景参照物的相对关系不突然改变，人物不要忽大忽小。

### 服装 logo 和细节一致性

每条提示词必须明确，但优先压缩成一句服装锁定块：

- 保持服装上的 logo、印花、纽扣、拉链、口袋、走线、装饰件 1:1 一致。
- 保持原大小、原位置、原形状、原比例。
- “不变形、不变大、不变小、不漂移、不新增”默认放进负面提示词；主提示词写“保持原大小、原位置、原形状、原比例”即可。
- “画面不要出现文字、字幕、水印、价格牌、额外 logo”不能误伤服装本身已有的 logo、印花和细节；服装已有内容必须保持。

### 服装质感一致性

服装质感必须锁死。默认合并进服装锁定块，只有面料是核心卖点或经常失败时才单独展开：

- 保持参考图/人物四视图中服装的同一面料材质、同一织物纹理、同一厚薄、同一垂感、同一光泽、同一透明度、同一弹性、同一硬挺度、同一褶皱位置、同一褶皱方向、同一走线纹理、同一边缘厚度、同一拼接结构。
- 人物动作时，面料动态只允许自然跟随身体轻微变化。
- 材质改变、纹理闪烁、褶皱乱跑、布料变塑料、布料变纸片、布料过度丝滑、布料忽厚忽薄等放进负面提示词。

针织、镂空、蕾丝、网眼、细孔、竖向纹理等高风险服装必须额外锁定透明度。除非用户明确要求透视/薄纱效果，否则默认写成“实心不透肉的针织面料，纹理只是表面织法和阴影，不形成透明孔洞，不露出肤色、内衣轮廓或身体轮廓”。不要只写“镂空纹理”或“轻薄感”，这会让模型把衣服生成成透视薄纱。

透明度修正必须写成通用服装规则，不能默认绑定某一款、某个颜色或某个 SKU。除非用户明确要求针对具体款号改写，否则使用“上装/下装/连衣裙/外套/服装主体/当前服装”等通用对象。示例：写“服装主体保持参考图中的原始厚薄和透明度”，不要写“蓝色上衣必须保持……”。如果用户提供具体款号和素材，才把通用对象替换成对应品类和颜色。

### 背景和构图一致性

有背景图或参考视频时，提示词必须锁定背景和构图：

- 保持统一背景、光线方向、色调、人物比例、场景纵深和手机拍摄质感。
- 背景中的门、窗、墙角、扶手、吊环、地砖线、地面反光、镜子边缘、沙发边缘等真实参照物要稳定。
- 不要让背景跳变、变成棚拍、变成虚假完美背景或过度高清广告质感。
- 如果背景由单独背景参考生成，直接写 `背景@`，不要写 `Image2`，也不要只给用户一段背景提示词。

### 真人自然感

默认加入这些低 AI 感细节，按画面选择合适项，不要全部堆满：

- 整体像真实抖音手机自拍视频，不要广告棚拍感
- 人物以中近景或大半身为主时，头顶留白较少，画面可裁到大腿附近，不要强行拉成全身远景
- 固定简单室内背景、普通墙面、窗边自然光或参考视频中的真实空场景，不要过度干净的商业棚拍背景
- 正常呼吸带来的胸肩轻微起伏
- 自然眨眼，眼神有细微移动
- 放松微表情，不假笑，不全程固定微笑；嘴角有轻微变化，眼神先看镜头再短暂偏开
- 轻微重心转移，站姿小幅调整
- 肩颈放松，手指自然弯曲
- 轻松随意的短视频手势：轻微摆手、手掌短暂靠近镜头、自然收回、肩膀和胯部小幅摆动
- 动作之间有短暂停顿，左右手不同步，节奏略有随机性，不要机械循环
- 自然走一两步、整理衣摆、轻扶包带、转头看向镜头
- 衣摆、袖口、发丝随动作轻微晃动
- 肤色白净自然，但保留鼻翼、眼下、嘴角、锁骨和肩颈的真实阴影，不要陶瓷皮、发光皮、过度磨皮
- 画面保留抖音手机视频的轻微压缩感和柔和边缘，人物清晰但不要过度高清，不要商业精修质感

不要写慢动作。动作应像手机真实拍摄的日常展示，节奏自然，幅度小，不能像机械转身或过度丝滑广告片。加入自然呼吸起伏、正常眨眼、眼神轻微移动、视线有明确落点、放松表情、正常动作速度、真实皮肤纹理和自然偏白肤色。

### 女装千川表达

优先选择这些画面重点：

- 版型轮廓、上身效果、显瘦/显高的视觉观感
- 面料垂坠、柔软度、挺括感、轻薄感或保暖感
- 走动时的衣摆动态、腰线、肩线、裙摆、裤型
- 通勤、约会、日常、出游、逛街、办公室、电梯口、街边、咖啡店等真实场景
- 细节近景：领口、袖口、腰部、裙摆、面料纹理、纽扣、拉链、搭配包鞋

卖点不明确时，从版型、面料、遮肉、百搭、场景感、上身利落、视觉显高显瘦中选择 1-2 个。文案用自然表达，如“视觉上更显利落”“日常好搭”“通勤也不费力”“上身不容易显拖沓”。避免绝对化承诺。

### 千川投流脚本强度

用户提到千川、投流、广告素材、跑量、测款、素材方向，或批量要多条提示词时，按投流素材而不是普通服装展示来写：

- **前 1-3 秒钩子**：必须是可见动作或场景阻力，例如电梯门快开前整理包带、出门前在玄关镜旁确认腰线、咖啡店门口转身避开人流、办公室玻璃门前停一步看侧身。不要只写“镜头推进展示上身”。
- **买家任务**：每条只选一个核心任务，例如省搭配、通勤利落、显比例、不显拖沓、适合空调房、夏天看起来干净、出门不用重新换衣服。
- **真实场景阻力**：必须有一个具体小问题推动动作，例如赶电梯、包带滑落、坐下后起身、路过玻璃反光、风吹裙摆、进办公室前整理袖口。阻力要轻，不要戏剧化。
- **卖点证明**：至少两段画面证明，优先用腰线、上衣针织纹理、领口/袖口、裙摆垂坠、包鞋搭配、走动时比例、坐站切换后的服装状态。不能只写“高级、好看、显瘦”。
- **结尾记忆点**：最后 2 秒要形成一个可被剪辑定格的画面，例如正面全身稳定、侧身腰线清楚、低头扣包链后抬眼、半步回头看镜头。
- **批量差异**：一次输出多条时，每条必须测试不同心理机制或买家任务；不得输出 10 条同质“走动展示”。

**千川开头钩子库**（每条素材选 1 个，先转成画面，再配后期文案）：

| 类型 | 适合素材 | 画面/文案公式 |
|---|---|---|
| 结果前置 | 卖点一眼可见，如显利落、显比例、裙摆垂坠 | 先把结果拍出来：我用一眼告诉你，为什么 XXX。 |
| 反常识 | 用户对版型、面料、搭配有误解 | 很多人以为 XXX，其实真正影响上身效果的是 XXX。 |
| 人群点名 | 明确人群，如小个子、通勤党、梨形身材、怕显拖沓 | 如果你是 XXX，这条一定要看完。 |
| 痛点代入 | 用户有具体穿搭尴尬 | 你是不是也遇到过这种情况？XXX。 |
| 错误提醒 | 常见错误搭配或选款误区 | 别再 XXX 了，你真正该看的是 XXX。 |
| 冲突对比 | 普通穿法 vs 处理后、静态 vs 走动、远景 vs 细节 | 普通人 XXX，会穿的人 XXX。 |
| 悬念缺口 | 有一个不直觉的小细节 | 我发现一个很容易被忽略的细节：XXX。 |
| 失败复盘 | 买错衣服、穿搭翻车、显拖沓经验 | 我见过太多 XXX，最后都输在 XXX。 |
| 成本提醒 | 当前问题会浪费钱、压身高、降低穿着率 | 你以为 XXX，其实正在让你 XXX。 |
| 标准建立 | 做选款判断、测款对比、讲面料/版型 | 判断 XXX 好不好，不要看 XXX，要看 XXX。 |

选型规则：强卖点直给用“结果前置 / 标准建立”；人群明确用“人群点名 / 痛点代入”；纠正误区用“反常识 / 错误提醒”；需要制造停留用“冲突对比 / 悬念缺口”；复盘买错或穿错用“失败复盘 / 成本提醒”。钩子必须能被前 1-3 秒画面证明，不能只靠字幕成立。

15 秒素材推荐使用 `0-2s 钩子 / 2-5s 场景阻力 / 5-9s 卖点证明一 / 9-12s 卖点证明二 / 12-15s 结尾记忆点`。如果一条里动作超过 5 个，优先删动作，不要把脚本塞满。

### 千川购买心理

写素材方向和文案时，先选 1 个主心理机制，不要一条素材塞满所有套路：

- **JTBD**：用户买的不是衣服本身，而是“显得更利落/更瘦/更有场景感/更省搭配时间/更不容易出错”。
- **即时收益**：前 1-3 秒优先呈现一眼能看懂的变化，如腰线、垂坠、遮肉、走动效果、上身利落感。
- **损失厌恶**：温和表达“不踩雷/不显拖沓/不压身高/不用担心搭配翻车”，避免制造焦虑或夸大承诺。
- **社会证明**：只有用户提供真实评价、销量、复购、达人反馈时才使用；不要编造“很多人都买”。
- **对比效应**：如果素材支持，可做静态到走动、普通站姿到整理后、远景到细节近景的对比，不要凭空写夸张 before/after。
- **降低后悔**：强调日常场景、通勤可穿、好搭、不挑场合、动作自然，让购买风险感觉更低。

开头钩子应短、具体、可被画面证明。优先写“这条裙子上身不容易显拖沓”这类可验证表达，少写“高级感拉满”“闭眼入”这种空话。

### 画面文字规则

不要让 Seedance 在视频画面里生成任何文字。默认必须明确写“禁止字幕、口播文字和屏幕文字”。提示词中不要要求字幕、价格、logo、水印、品牌字样、吊牌文字、广告标语、大字报、手机界面文字、屏幕 UI 或任何可读文字。口播可以作为声音/口播音频生成，也可以作为后期配音参考；字幕、价格和利益点只放在文案建议或后期剪辑，不要作为画面文字生成。

## Negative Prompt

负面提示词要压缩保留，只覆盖硬错误。不要输出裸名词列表，必须写成“避免项（以下全部不要生成，不是画面内容）：避免生成……”这种明确禁止语气。主提示词尽量少写否定句；高风险细节先在正向锁定里写清楚，再在避免项里补硬错误。默认包含：

```text
脸型漂移、眼神异常、五官漂移、手指畸形、身体比例异常、人物站位漂移、脚底落点漂移、人物忽大忽小、服装颜色变化、服装版型变化、服装质感变化、面料纹理变化、衣服变透明、薄纱化、透肉、肤色透出、内衣轮廓、身体轮廓透出、镂空孔洞扩大、logo变形、logo大小变化、logo位置漂移、印花变形、纽扣变形、拉链变形、口袋变形、走线变形、背景跳变、镜头突变、动作卡顿、字幕、口播文字、屏幕文字、屏幕UI文字、手机界面文字、水印、价格牌、额外logo。
```

图生视频时补充：

```text
避免改变参考图人物身份、脸型、五官、眼型、眼距、眼神气质、发型、妆容、服装、服装质感、logo、印花、配饰、道具位置、人物站位、主体比例和主要构图。
```

## Copy Rules

千川文案建议要像真实短视频剪辑可用，不要像广告模板。默认包含：

- 开头钩子：标明钩子类型 + 1 句适合前 1-2 秒吸引停留的话，例如“标准建立：判断一条裙子显不显拖沓，不要只看腰线，要看走动时裙摆怎么落。”
- 心理机制：说明本条素材主打 JTBD、即时收益、损失厌恶、社会证明、对比效应或降低后悔中的哪一个。
- 后期文案建议：2-4 句，短、顺口，只供剪辑后期添加；不要写进 Seedance 画面生成指令。
- 卖点表达：围绕 1-2 个核心卖点，不夸大。
- 剪辑建议：说明文案适合加在开头、细节近景、走动展示或结尾。

不要让文案过硬广。除非用户明确要求强转化，否则默认“原生种草 + 轻广告感”。后期文案可以使用钩子句，但 Seedance 画面提示词里仍然禁止生成字幕、口播文字和屏幕文字。
